This repository contains my current Emacs configuration, designed to suit my preferences. However, parts of it may be useful to you, especially if you are a JavaScript, Python, or C/C++ developer.

* How to Install

** Prerequisites

This configuration depends on a number of external system packages.

 - Git for [[https://magit.vc/][magit]] and [[https://github.com/emacsorphanage/magit-gerrit][magit-gerrit]]
 - Cscope for [[https://github.com/dkogan/xcscope.el][xcscope]]
 - Node.js / NPM for [[https://github.com/yasuyk/web-beautify][web-beautify]], [[https://github.com/typescript-language-server/typescript-language-server][typescript-language-server]], and [[https://github.com/ternjs/tern][tern]]
 - The silver searcher for [[https://github.com/Wilfred/ag.el][ag]] and [[https://github.com/emacsorphanage/helm-ag][helm-ag]]
 - Python / pylsp for [[https://gitlab.com/python-mode-devs/python-mode][python-mode]] and [[https://github.com/pythonic-emacs/anaconda-mode][anaconda-mode]]
 - Markdown for [[https://jblevins.org/projects/markdown-mode/][markdown-mode]] preview
 - JRE + PlantUML for [[https://github.com/skuro/plantuml-mode][plantuml-mode]]

Thanks to the ~:ensure-system-package~ feature of use-package, most of these are installed automatically on first launch. Some require sudo, so you may be prompted for your password.

** System Environment

Emacs 30 or newer is required. Several features rely on built-ins introduced in Emacs 29 (e.g. ~use-package~), and =config/claude.el= uses use-package's ~:vc~ keyword, which is only available in Emacs 30 and newer.

** Compiler

The C/C++ header path detection is tailored for GCC. If you use a different compiler, automatic include path discovery may not work correctly.

** Installation

#+BEGIN_SRC sh
  mv ~/.emacs.d ~/.emacs.d.old
  git clone --recursive git@github.com:zeph1e/.emacs.d.git
#+END_SRC

Remaining packages are installed automatically on first launch.

* What's in here

While using Emacs for quite some time, I've accumulated various bits of code to enhance convenience. Here I will cover the parts that have been implemented. I hope it serves as a helpful resource, either for using this configuration as is or as inspiration for building your own.

* Architecture

The configuration is structured around a few top-level entry points:

 - =init.el= — Entry point. Sets up locale, UI, the Emacs server, backup directories, =package.el= / =use-package=, then loads =plugins/= and =config/=.
 - =config/= — One =.el= file per concern (e.g., =helm.el=, =lsp.el=, =git.el=). All files are loaded at startup and byte-compiled when Emacs quits.
 - =plugins/= — Manually managed packages not on ELPA/MELPA. Byte-compilation and autoload generation happen once on first load; a =.installed= sentinel file prevents re-processing.
 - =workaround.el= — Loaded early; contains targeted fixes for upstream bugs.
 - =custom.el= — Auto-generated by =M-x customize=; excluded from version control.

All custom keybindings live in ~my:global-key-map~, a minor mode that always takes precedence over major and minor mode keymaps. Bindings go into ~:map my:global-key-map~ inside ~use-package~ ~:bind~ blocks — never ~global-set-key~ directly.

* Basic Configuration

** UI & Locale

The UI starts minimal: no toolbar, no menu bar, no startup screen. ~global-hl-line-mode~ highlights the current line. The fill column is set to 80, and ~display-fill-column-indicator-mode~ is enabled in all ~prog-mode~ buffers. Locale is set to Korean with UTF-8 encoding.

Fonts are configured only in graphical sessions. [[https://fonts.google.com/specimen/Nanum+Gothic+Coding][Nanum Gothic Coding]] covers the Korean character range (U+1100–U+FFDC), while =Lucida Console= is used for everything else. The frame is maximized on launch.

** Emacs Server

The server is started automatically on launch, storing its type (=daemon=, =gui=, or =tty=) as a process property. A helper shell script, =misc/edit=, acts as a smart =emacsclient= wrapper: it reuses a running instance if one exists, choosing =-c= or =-t= depending on whether the server is graphical or terminal. Setting =$EDITOR= to =edit= integrates Emacs into the shell seamlessly.

** Backups & Auto-saves

All auto-generated files are redirected into hidden subdirectories under =~/.emacs.d= (=.backups=, =.auto-saves=, =.lock-files=). This keeps the filesystem clean and prevents Tramp sessions from hanging on remote auto-save writes.

** Package System

The built-in =package.el= is the package manager, with GNU ELPA, NonGNU ELPA, and MELPA configured as archives. [[https://github.com/jwiegley/use-package][use-package]] is built-in from Emacs 29 and is the canonical declaration form. Packages not on any archive are pulled in with use-package's =:vc= keyword (which delegates to =package-vc-install=); this requires Emacs 30 or newer.

** Default Minor Modes

Minor modes applied to every buffer by category:

| Category | Minor modes |
|----------+-------------|
| All ~prog-mode~ | ~display-line-numbers-mode~, ~my:whitespace-mode~, ~flyspell-prog-mode~, ~display-fill-column-indicator-mode~, ~goto-address-prog-mode~ |
| All ~text-mode~ | ~display-line-numbers-mode~, ~my:whitespace-mode~, ~visual-line-mode~, ~flyspell-mode~, ~goto-address-mode~ |

~global-hl-line-mode~ is suppressed in shell, eshell, and term buffers where the highlight is visually disruptive.

* UX Configuration

** Look & Feel

*** Theme

I've used the Night Eighties variant of [[https://github.com/chriskempson/tomorrow-theme][Tomorrow Theme]] for most of my time with Emacs. The color contrast is well-balanced without being overly saturated. It is loaded from =plugins/color-theme-tomorrow=. The only adjustment I made is setting the ~highlight~ face background to =DeepSkyBlue4= for better text contrast.

*** Mode Line

#+HTML: <img src="https://github.com/zeph1e/.emacs.d/blob/master/misc/res/modeline.png?raw=true">

I customized the mode line rather than using Powerline or similar packages, because I wanted only the information I actually read: buffer size, modification flag, buffer name (red background when read-only), line:column, input method indicator, major mode, a [[https://github.com/TeMPOraL/nyan-mode][Nyan Cat]] progress bar, scroll percentage, and the vc-mode branch. Everything is in =config/theme.el=. If you'd like to do the same, the [[https://emacs-fu.blogspot.com/2011/08/customizing-mode-line.html][Emacs-fu article on mode-line customization]] is a good starting point.

*** Header Line

The header line displays the full path of the editing file, truncating the directory portion with =[...]= when it is too wide for the window. The code is taken from [[https://www.emacswiki.org/emacs/HeaderLine][Emacs Wiki]], in =config/theme.el=.

** Helm & Company

I initially used [[https://www.gnu.org/software/emacs/manual/html_mono/ido.html][ido]] and [[https://github.com/auto-complete/auto-complete][autocomplete]], but later switched to [[https://github.com/emacs-helm/helm][helm]] and [[https://company-mode.github.io/][company]], which I've been using ever since.

*** Helm

[[https://github.com/emacs-helm/helm][Helm]] replaces ido for all interactive selection, configured in =config/helm.el=. The most useful customization is ~my:helm-find-files-backward-dwim~: pressing =<backspace>= in a file path deletes one directory level rather than one character, making navigation of deep paths much faster.

| Key | Command |
|-----+---------|
| =M-x= | ~helm-M-x~ |
| =C-x C-f= | ~helm-find-files~ |
| =C-x b= | ~helm-mini~ |
| =M-y= | ~helm-show-kill-ring~ |
| =M-r= | ~helm-occur~ |
| =M-R= | ~helm-grep-do-git-grep~ |
| =C-h a= | ~helm-apropos~ |
| =C-h b= | ~helm-descbinds~ |
| =C-x C-p= / =C-x p= | Projectile command map (helm-projectile) |

[[https://github.com/emacs-lsp/helm-lsp][helm-lsp]] replaces ~xref-find-apropos~ with a helm-powered workspace symbol search when LSP is active.

*** Company

[[https://company-mode.github.io/][Company]] is enabled globally. Completion is triggered with =C-M-;=; candidates are navigated with =C-p= / =C-n= and paged with =C-v= / =M-v=.

A custom ~my:company-ispell~ wrapper activates spell-check completion only inside comments, docstrings, and string literals by inspecting the face at point, keeping it out of regular code tokens.

For C/C++, ~company-c-headers~ is configured with compiler include paths detected at runtime by invoking =gcc -x c -E -v -= and parsing the output. Qt headers are picked up automatically when =qmake= is available. Mode-specific backends:

| Mode | Backends |
|------+----------|
| web-mode, sgml-mode | company-web-html, company-css, company-tern |
| css-mode, less-css-mode | company-css |
| js-mode | company-tern |
| c-mode, c++-mode, objc-mode | company-c-headers |
| python-mode | company-anaconda |

* Editing

** Multi-editing

[[https://github.com/victorhge/iedit][iedit]] (=C-M-#=) edits all occurrences of a symbol within its scope simultaneously — ideal for quick in-buffer renaming. [[https://github.com/magnars/multiple-cursors.el][multiple-cursors]] is better suited for structured multi-line edits:

| Key | Command |
|-----+---------|
| =M-.= | Mark next like this |
| =M-,= | Mark previous like this |
| =M-/= | Mark all like this |
| =M-?= | Edit lines in region |

** Custom Editing Commands

A collection of small ergonomic commands in =config/editor.el=:

| Key | Command |
|-----+---------|
| =M-@= | Mark word at point |
| =M-#= | Mark symbol at point |
| =M-SPC= | Jump to first non-blank character on line |
| =C-o= | Open new line above |
| =M-o= | Insert new line below |
| =<f12>= | Toggle buffer read-only |

~kill-line~ is advised to clean up leading whitespace after the kill, preventing orphaned indentation.

** Whitespace

~my:whitespace-mode~ wraps the built-in ~whitespace-mode~ with mode-aware style. In ~prog-mode~ it highlights trailing whitespace, long lines (beyond fill-column), and tabs. In ~text-mode~ it omits the long-line indicator to avoid noise in prose.

** Undo / Redo

[[https://www.emacswiki.org/emacs/RedoPlus][redo+]] (=plugins/redo+=) adds conventional linear redo on top of Emacs' undo tree.

* Window Management

All bindings are in =config/window.el=.

| Key | Action |
|-----+--------|
| =C-<arrow>= | Move focus to adjacent window |
| =S-<arrow>= | Swap current window with adjacent ([[https://github.com/konstantinblaesi/windswap][windswap]]) |
| =C-S-<arrow>= | Copy buffer to adjacent window ([[https://github.com/dholm/windcopy][windcopy]]) |
| =M-<arrow>= | Move focus across frames ([[https://github.com/tlikonen/framemove][framemove]]) |
| =C-x \= / =C-x -= | Split horizontally / vertically |
| =C-x |= / =C-x _= | Split and move into new window ([[https://github.com/nex3/windsplit][windsplit]]) |
| =C-x x= | Close current window |
| =C-{= / =C-}= | Shrink / enlarge window horizontally |

windcopy is handy for duplicating a buffer into a split view without disturbing the other window. framemove hooks into windmove so =C-<arrow>= wraps across frames transparently. Scrolling is patched so =C-v= / =M-v= at the buffer boundary moves to the end / beginning rather than signaling an error.

* Build

=<f7>= calls ~recompile~ when a previous compilation buffer exists, or falls back to the interactive ~compile~ prompt otherwise. =C-<f7>= always opens the interactive prompt. Both are configured in =config/compile.el=.

* External File Viewer

=config/fileviewer.el= routes external file opens and URL navigation to the host's default viewer, with awareness of where Emacs is running:

 - *WSL* — uses =wslview= so files and URLs open in the Windows host.
 - *SSH-remote* (=$SSH_CLIENT= set) — falls back to local =xdg-open= when a graphical display is available; otherwise the operation is skipped.
 - *Local* — uses =xdg-open=, =gnome-open=, =kde-open=, or =open=, whichever is found.

In dired, =V= invokes the resolved viewer on the file at point; remote files are first copied to a temp file via Tramp. ~browse-url-browser-function~ is wired to the same opener so external links land in the host browser, and =mailcap= viewer entries on WSL are rewritten to go through =wslview=.

*** SSH-remote setup

When Emacs runs on a remote server, ~browse-url~ resolves URLs by =ssh=-ing back to the workstation (the host found in =$SSH_CLIENT=) and invoking =xdg-open= there. For this to work non-interactively:

 1. *Reverse SSH access* — the workstation must run =sshd=, and the remote-host user's public key must be installed in =~/.ssh/authorized_keys= on the workstation, so password / passphrase prompts don't block Emacs.
 2. *Username mapping* — =ssh= defaults to the current user on the remote host. If your workstation login differs from your login on the remote host, add an entry to =~/.ssh/config= on the *remote* host:

    #+BEGIN_SRC conf
      Host <workstation-host-or-ip>
          User <workstation-username>
          IdentityFile ~/.ssh/id_ed25519
    #+END_SRC

    The =Host= pattern must match the IP returned by =$SSH_CLIENT= (the first whitespace-separated field) — typically the workstation's outbound address as seen from the server.
 3. *Host key trust* — the workstation's host key must already be in the remote host's =~/.ssh/known_hosts=, otherwise the first connection blocks on a confirmation prompt. Connect once manually (=ssh <workstation>=) before relying on URL handling.

For dired =V= on the SSH-remote case, no reverse SSH is used — Emacs runs the *local* =xdg-open= and relies on graphical display forwarding (X11 / Wayland) to render on the workstation. Launch =emacs= with a working =$DISPLAY= (i.e. via =ssh -X= / =-Y=) for this branch to take effect.

* Snippets & Note Taking

[[https://github.com/joaotavora/yasnippet][YASnippet]] is enabled globally. Custom snippets go in the =snippets/= directory.

[[https://jblevins.org/projects/deft/][Deft]] (=M-g d= or =C-x RET C-d=) provides fast plain-text note browsing. Notes live in =~/Documents/deft= with recursive search, defaulting to the =.org= extension.

* Org Mode

Org Babel language support is configured in =config/org.el=. Language availability is determined at startup by checking for the relevant executables (=gcc=, =g++=, =python=, =gnuplot=, =java=, etc.). PlantUML blocks use the jar configured by =plantuml-mode=. [[https://github.com/hniksic/emacs-htmlize][htmlize]] is included for syntax-highlighted HTML export.
